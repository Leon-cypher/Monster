## ‣ 玩家查詢（網站功能）

玩家在官網輸入自己的 **玩家 ID** 後，可查詢以下資訊：

### 查詢結果顯示內容

| 欄位 | 說明 |
|------|------|
| 目前點數 | 目前累積的有效點數（大數字顯示） |
| 可抽牌次數 | `floor(點數 / 10)`，代表還可以去兌換幾次 |
| 已抽次數 | 歷史上已至櫃台兌換過幾次 |
| 中獎紀錄 | 列出曾獲得 🥇大獎 或 🥈普獎 的記錄，顯示階段與獎項（未中獎記錄不顯示） |

### 查詢狀態說明

| 狀態 | 說明 |
|------|------|
| 查詢中 | 正在向資料庫讀取 |
| ⚠️ 請輸入玩家 ID | 空白送出 |
| ⚠️ 找不到此玩家 ID | ID 尚未建立（尚未消費登記）|
| ⚠️ 查詢失敗 | 網路或資料庫異常 |

> 中獎紀錄由後台「輸入抽獎結果」功能寫入，玩家端即時反映。

## ‣ 後台操作說明（店員）

後台網址：`/admin`（不對外公開，請勿分享連結）

---

### 1. 玩家查詢

1. 在「玩家查詢」面板輸入玩家 ID，按「查詢玩家」或 Enter
2. **找到玩家**：顯示目前點數、可抽牌次數、已抽次數
3. **找不到玩家**：出現「建立新玩家」按鈕 → 點擊即可建立（首次消費時使用）

---

### 2. 加點 / 扣點

| 操作 | 說明 |
|------|------|
| 加點 | 玩家消費 BUYIN / REBUY / ADDON 時，依金額加對應點數 |
| 扣點 | 玩家兌換抽牌時扣 10 點（系統自動累計已抽次數） |

**操作步驟：**
1. 輸入玩家 ID
2. 選擇操作類型（加點 / 扣點）
3. 點選「金額快選」按鈕（自動帶入點數與操作類型），或手動輸入點數
4. 按「確認操作」

> ⚠️ 扣點時若點數不足，系統會擋下並提示目前剩餘點數

**金額快選對照：**

| 快選按鈕 | 操作 | 點數 |
|----------|------|------|
| $1,200→1pt | 加點 | +1 |
| $3,400→2pt | 加點 | +2 |
| $6,600→3pt | 加點 | +3 |
| $11,000+→5pt | 加點 | +5 |
| 兌換-10pt | 扣點 | -10 |

---

### 3. 輸入抽獎結果

玩家持牌來兌換獎品時使用：

1. 輸入玩家 ID
2. 輸入手牌（如 `A♠`、`K♥`，可留空）
3. 選擇獎項結果：
   - 🥇 **大獎**（鐵支以上）→ $30,000
   - 🥈 **普獎**（葫蘆）→ 均分 $20,000
   - ❌ **無**（未中獎）
4. 按「送出結果」

> 結果會即時寫入資料庫，玩家查詢端可同步看到中獎記錄

---

### 4. 即時統計

切換頁面頂部的「階段選擇器」可查看各階段數據：

| 欄位 | 說明 |
|------|------|
| 🥇 大獎 | 本階段大獎得主人數 |
| 🥈 普獎 | 本階段普獎得主人數 |
| 👥 總參與 | 本階段總抽獎記錄數 |
| ❌ 未中獎 | 本階段未中獎人數 |
| 普獎每人可分金額 | 自動計算 $20,000 ÷ 中獎人數（保底 $500，超過 40 人改發現金+點數） |

---

### 5. 抽獎記錄

所有抽獎紀錄的歷史清單，支援篩選：

- **依階段篩選**：全部 / 第一～四階段
- **依結果篩選**：全部 / 大獎 / 普獎 / 未中獎

資料來源為 Firebase 即時同步，無需重整頁面。
